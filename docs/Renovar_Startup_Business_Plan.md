# I. Executive Summary

*(Write after completing the rest of the plan; below is a completed version for Renovar.)*

**Business idea (1–2 sentences)**  
Renovar is a technical solution that helps European insurers migrate data from legacy systems to modern platforms and expose legacy capabilities through modern APIs, reducing cost, compliance risk, and time-to-market for digital and S/4HANA initiatives.

**Product/service and problem solved**  
Renovar provides automated data mapping, ETL, AI-driven validation, and RESTful API wrappers for legacy insurance applications. It solves fragmented data, high maintenance cost (15–20% of IT budget), manual rework (€2–5M/year), and lack of real-time APIs that delay digital channels and increase churn. Target customers are mid-to-large insurers in the UK, Germany, France, Netherlands, and Italy (GDPR, Solvency II).

**Goals (1, 3, 5 years)**  
- **Year 1:** Pilot with 1–2 insurers (e.g., one business line each); validate product-market fit; achieve first paying customers and referenceable case studies.  
- **Year 3:** 10–15 European insurer customers; recurring revenue from licensing and support; partnerships with 2–3 SIs or SAP-focused consultancies.  
- **Year 5:** Recognized brand in European insurance data migration and API modernization; optional expansion to adjacent verticals or geographies.

**Target market**  
Mid-to-large European insurance companies (1M+ policies) across Personal (auto, home), Commercial (liability, property), P&C, Life, and Non-Life (health, travel), operating under GDPR and Solvency II.

**Competition and differentiation**  
Competitors include IBM InfoSphere, Oracle GoldenGate, Microsoft Azure Data Factory, Accenture, Informatica, Capgemini, Ispirer, Fadata, Talend, Airbyte, Intellias. Renovar differentiates through: SAP ABAP–native design (ideal for SAP-heavy insurers), 20–30% lower TCO, 3–6 month pilot vs. 9–12 months for custom programmes, insurance-line agnostic scope without full core replacement, and built-in API wrappers for real-time use cases.
