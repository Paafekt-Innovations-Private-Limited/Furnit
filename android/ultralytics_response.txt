For implementing Lanczos resampling on Android for a 3D reconstruction pipeline like SHARP:

1. Implementation: OpenCV Imgproc.resize with INTER_LANCZOS4 is most robust (native/NEON). Custom Kotlin kernels can be slow for 2000px; we use a Kotlin Lanczos3 with a switch so you can fall back to bilinear if needed.

2. Kernel: Lanczos3 (6x6) recommended for best quality/speed tradeoff for ~2000->1536 downscale.

3. INT8: Lanczos can introduce slight ringing; clamp output to 0..255 to match bilinear range. If quality degrades, set USE_LANCZOS_RESIZE=false.
