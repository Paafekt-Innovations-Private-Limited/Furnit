In our Android "Furniture Fit" overlay we display a furniture mask (or photo) over a 3D room. The user can pinch-to-zoom to scale the overlay (furnitureScale from 0.3x to 3x). When the content is scaled to 0.5x, it is positioned wrongly and not in the middle of the view.

Current logic (Kotlin):

  • We use ScaleGestureDetector for pinch; furnitureScale *= detector.scaleFactor, clamped to [0.3, 3].
  • Drawing: baseScaleX = width / bmp.width, baseScaleY = height / bmp.height. Scaled size = bmp.size baseScale furnitureScale.
  • centerOffsetX = (width - scaledWidth) / 2 (intended to center horizontally).
  • floorOffsetY = height * 0.35f (place content in lower part of screen).
  • matrix.setScale(baseScaleX furnitureScale, baseScaleY furnitureScale); matrix.postTranslate(centerOffsetX + translateX, floorOffsetY + translateY).
  • User can drag to translate (translateX, translateY).

Problem: at 0.5x scale the content is not in the middle. We need the overlay to stay centered (or correctly positioned) at any scale.

Ultralytics AI

While this is an Android UI task, maintaining spatial consistency during scaling is similar to how Ultralytics YOLO handles coordinate scaling when resizing images.

1. Scaling Focus Point

Yes, ScaleGestureDetector scales around the focus point (the midpoint between fingers). If you apply matrix.setScale followed by a static centerOffsetX, the content will "jump" because the gesture's focal point isn't accounted for. To keep the content centered during a pinch, you must scale relative to the view's center (or the gesture focus) and adjust the translation accordingly.

2. Center Offset Formula

Your formula centerOffsetX = (width - scaledWidth) / 2 is mathematically correct for calculating the top-left coordinate to center a box of scaledWidth within a container of width. If the content is not centered at 0.5x, the issue likely stems from:

  • Pivot Point: matrix.setScale(sx, sy) scales from (0,0).
  • Translation Order: Ensure you aren't double-applying scale to your translation values.

3. Recommended Fix

To keep the furniture centered at any scale, use the matrix.postScale method with a pivot point. This automatically handles the translation shift required to keep a specific point fixed.

Corrected Logic (Kotlin):

// 1. Calculate base scale to fit screen width/height initially
val baseScale = min(width / bmp.width.toFloat(), height / bmp.height.toFloat())
val totalScale = baseScale * furnitureScale

matrix.reset()
// 2. Scale around the BITMAP center
matrix.postScale(totalScale, totalScale, bmp.width / 2f, bmp.height / 2f)

// 3. Move the BITMAP center to the target SCREEN center
val screenCenterX = width / 2f
val screenCenterY = height * 0.35f // Your desired floor offset
matrix.postTranslate(screenCenterX - (bmp.width / 2f), screenCenterY - (bmp.height / 2f))

// 4. Apply user drag offsets
matrix.postTranslate(translateX, translateY)

Key Improvements:

  • Pivot Scaling: Scaling around bmp.width / 2f ensures the expansion/contraction happens from the object's own center.
