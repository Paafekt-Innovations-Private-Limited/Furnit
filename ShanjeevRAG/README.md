# ShanjeevRAG

A **RAG (Retrieval Augmented Generation)** for the **insurance domain**. Add your own documents, train iteratively, and run queries locally. Share the repo (e.g. via GitHub) so others (e.g. Shanjeev) can clone and run the same RAG on their laptop.

## What it does

- **Ingest:** Add PDF, DOCX, or TXT files (policies, guidelines, regulations). The pipeline chunks text, embeds with a local model, and stores vectors in ChromaDB.
- **Query:** Ask questions in natural language; the system retrieves relevant chunks and optionally generates an answer using an LLM (OpenAI or Perplexity).
- **Iterative training:** Add more documents to `data/` and run `python ingest.py` again. The vector DB is updated; no full retrain from scratch required (use `--reset` only when you want to rebuild from zero).

## For Shanjeev (when shared on GitHub)

Everything runs on **his laptop** after he clones the repo. He can use Perplexity as the answer engine by getting a Perplexity API key and using `--llm perplexity` (the app sends the RAG context to Perplexity’s API; he does **not** use the Perplexity website for this).

1. **Clone** the repo (replace with the real GitHub URL):
   ```bash
   git clone https://github.com/YOUR_USERNAME/ShanjeevRAG.git
   cd ShanjeevRAG
   ```
2. **Create a venv and install** dependencies:
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate   # Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```
3. **Build the RAG** from the documents in `data/` (first run downloads the embedding model):
   ```bash
   python ingest.py
   ```
4. **Query** (no API key needed; he just sees retrieved chunks):
   ```bash
   python query.py "What is excess in motor insurance?"
   ```
5. **Use Perplexity for answers** (optional):
   - Get an API key from [Perplexity → Settings → API](https://www.perplexity.ai/settings/api).
   - Copy `.env.example` to `.env` and add:
     ```bash
     PERPLEXITY_API_KEY=pplx-...
     ```
   - Run:
     ```bash
     python query.py "What is excess in motor insurance?" --llm perplexity
     ```
   Answers are generated by Perplexity using **only** the RAG context (web search is disabled), so they stay grounded in the repo’s documents.

   **Nothing to set in Perplexity.** Shanjeev does not type any keyword or change any setting in the Perplexity website or app. This project calls Perplexity’s API with the RAG context in the prompt and with web search turned off, so the model answers from the retrieved docs only, not from its normal search/LLM behaviour.

   **Perplexity model selection:** The project uses a default model (`sonar`) defined in `config.yaml` under `perplexity_model`. Shanjeev can change it there, or override per run with `--perplexity-model` (e.g. `sonar-pro`, `sonar-deep-research`, `sonar-reasoning-pro`). No need to pick a model inside Perplexity’s UI.

The repo includes `data/sample_insurance_terms.txt`, so clone + ingest + query works without adding any files.

### How Shanjeev trains the RAG (only in the project, not in Perplexity)

**Training** = adding knowledge so the RAG can retrieve it. That happens **only in this project**, by ingesting documents. Perplexity is not used for training; it only generates answers from the context we send it.

To train (add more knowledge):

1. Put new documents (PDF, DOCX, or TXT) into `data/` (or a subfolder).
2. Run:
   ```bash
   python ingest.py
   ```
   New chunks are added to the vector DB. Next time he queries (with or without `--llm perplexity`), the RAG will use this new content.

To **rebuild from zero** (e.g. after deleting or reorganising docs in `data/`):

```bash
python ingest.py --reset
```

To ingest specific files or folders without putting them in `data/`:

```bash
python ingest.py path/to/policy.pdf path/to/guidelines/
```

## Quick start (clone and run)

### 1. Clone the repo

```bash
git clone <your-repo-url> ShanjeevRAG
cd ShanjeevRAG
```

### 2. Create a virtual environment and install dependencies

```bash
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Ingest documents (first time)

The project includes a small sample file in `data/`. To build the RAG:

```bash
python ingest.py
```

The first run will download the embedding model (~80MB); then it chunks the files in `data/` and builds the vector DB in `vector_db/`. You’ll see “Ingest complete” when done.

### 4. Query the RAG

```bash
python query.py "What is excess in motor insurance?"
python query.py "Explain Solvency II" --top-k 5
```

Answers are based on the retrieved chunks from your ingested documents. No API key is needed for this.

### 5. (Optional) Generated answers with an LLM

If you want a single natural-language answer instead of raw chunks:

1. Copy `.env.example` to `.env` and set one of:
   - `OPENAI_API_KEY=sk-...` for OpenAI
   - `PERPLEXITY_API_KEY=pplx-...` for Perplexity (answer from your RAG context only; web search is disabled)
2. Run:

```bash
python query.py "What is excess in motor insurance?" --llm openai
# or
python query.py "What is excess in motor insurance?" --llm perplexity
```

## Iterative training (adding more knowledge)

Training happens **in the project only** (ingest documents into the vector DB). OpenAI and Perplexity are not used for training; they only generate answers from the retrieved context.

1. Add new insurance documents (PDF, DOCX, TXT) into `data/` (or into a subfolder).
2. Run ingest again:

```bash
python ingest.py
```

New chunks are **added** to the existing collection. To **replace** everything and rebuild from scratch:

```bash
python ingest.py --reset
```

You can also ingest specific paths without putting them in `data/`:

```bash
python ingest.py path/to/new_doc.pdf path/to/folder/
```

## Configuration

Edit `config.yaml` to change:

- `data_dir` / `vector_db_path` – where to read documents and store the DB
- `chunk_size` / `chunk_overlap` – chunking for retrieval
- `embedding_model` – default `all-MiniLM-L6-v2` (runs locally; no API key)
- `perplexity_model` – when using `--llm perplexity`, which model to use (default `sonar`; e.g. `sonar-pro`, `sonar-deep-research`, `sonar-reasoning-pro`). Can be overridden by `--perplexity-model` on the command line.

## Requirements

- Python 3.10+
- ~500MB disk for the default embedding model and dependencies

No API keys are required for ingest and retrieval-only query. Optional LLM: set `OPENAI_API_KEY` or `PERPLEXITY_API_KEY` in `.env` for `--llm openai` or `--llm perplexity`.
